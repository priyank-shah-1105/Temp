Resynchronizing an OS image repository

Use the resynchronize option to restore the OS image from the database after a backup and restore.

If an OS image was uploaded as a part of an ISO file, you need to resynchronize the OS image repository from the OS Image Repositories tab. However, if the OS was uploaded as a part of an RCM ZIP file, navigate to the Manage RCM Versions tab and resynchronize the RCM ZIP file there. The following procedure provides steps for performing the resynchronization from the OS Image Repositories tab.

To resynchronize the OS image repository for an OS image that was uploaded as part of an ISO file:

  1. On the OS and RCM Repositories page, click OS Image Repositories tab.

  2. From the Available Actions drop-down menu, click Resynchronize for a repository in an Error state.

    The Resynchronize OS Repository page is displayed.

  3. Enter the user credentials and click Test Connection to test the network connection.

    NOTE: You cannot edit the Source Path and Filename.

  4. Click Resynchronize.

    The repository state changes to Copying state.
