Resources

A resource is a physical and virtual datacenter object that VxFlex Manager interacts with including but not limited to nodes, network switches, VM managers (for example, VMware vCenter), and element managers (for example, OpenManage Enterprise).

The Resources page displays detailed information about all the resources and node pools that VxFlex Manager has discovered and inventoried. You can perform various operations from the All Resources and Node Pools tabs.

NOTE: It might take few minutes to display the discovered resources every time you run the inventory, depending upon the number of resources.

The All Resources tab displays the following information about the resources discovered and managed in VxFlex Manager:

Field nameDescription
Health

Indicates the resource health.

For more information about the resource state, see the Resource health status.

RCM Compliance

Indicates if the resource firmware and software compliance state is Compliant, Non-compliant, Update required, or Update failed.

Compliance is determined by the firmware/software version for the selected resource based on the default Release Certification Matrix (RCM). Click the compliance status to view the Release Certification Matrix (RCM) Compliance Report.

For more information about the compliance status, see the Resource firmware compliance status.

OS Hostname

Indicates the resource host name.

NOTE: VxFlex Manager retrieves the host name value from iDRAC and not the operating system. If the host name field is not updated in iDRAC, an incorrect value might display in VxFlex Manager. Certain operating systems require additional packages to be installed for iDRAC to update the correct host name.
Resource Name

Indicates the name of the resource.

Management IP

Indicates the resource IP address. Click the IP address to open the Element Manager.

Asset/Service Tag

Indicates the resource Service Tag.

Deployment Status

Indicates if the resource deployment status is In Use, Not in use, Available, Updating resource, or Pending Updates.

Click the deployment status to view service details.

Managed State

Indicates the resource state.

Model

Indicates the resource manufacturer name and model number.

To filter the resources that display, click the Toggle Filters icon on the Resource page.

  • To filter based on the resource type, select one of the following resource types from Resource Type drop-down list:
    • All
    • Element Manager
    • Nodes
    • Switches
    • VM Manager
    • VxFlex OS Gateway
  • To filter by health status, select one of the following statuses from the Health drop-down list:
    • All
    • Healthy
    • Warning
    • Critical
    • Unknown
  • To filter by resource state, select a Resource State option of Managed, Unmanaged, or Reserved.

On the Resources page, you can also:

  • Click Discover to discover new resources. For more information about discovering the new resources, see Discovering Resources.
  • Click Remove to remove the resource from VxFlex Manager. For more information about removing the resources, see Removing Resources.
  • Click Export All to export all resource details to a .csv file.
  • To manually run an inventory operation on a resource and update VxFlex Manager with the latest resource details, select a resource and click Run Inventory. After running the inventory, you can perform an Update Inventory on any service that requires the updated resource data.
  • Select one or more resources and click Update Resources to update the firmware and software of the resources.
  • From the right pane, you can perform the following actions:
    1. To view the detailed information about the resource, click View Details.
    2. Click the link corresponding to RCM Compliance to view the firmware and software compliance report.
    3. View the iDRAC power status.
