Removing a service

You can remove a service that is no longer required. VxFlex Manager supports two types of removal. You can delete the entire service, which deletes the deployment information and also makes any required configuration changes for components associated with the service. Alternatively, you can remove just the deployment information for the service without making any configuration changes to the deployed components.

Standard users can delete only the services that they have deployed.

To remove a service, perform the following steps:

  1. On the menu bar, click Services.

  2. On the Services page, click the service, and in the right pane, click View Details.

  3. On the Service Details page, in the right pane, under Service Actions, click Remove Service.

  4. In the Remove Service dialog box, Select the Service removal type:

    • Delete Service makes configuration changes to the nodes, switch ports, Virtual Machine Managers, and VxFlex OS to unconfigure those components. In addition, it returns the components to the available inventory.
    • Remove Service removes deployment information, but does not make any configuration changes to the nodes, switch ports, Virtual Machine Managers, and VxFlex OS. In addition, it returns the components to the available inventory.

  5. If you choose Remove Service, perform the following steps:

    1. If you want to keep the nodes in the inventory, select Leave nodes in VxFlex inventory and set state to and select the state:

      • Managed
      • Unmanaged
      • Reserved

    2. If you want to remove the nodes, select Remove nodes from the VxFlex inventory.

    3. Click Remove.

  6. If you choose Delete Service, perform the following steps:

    1. Select Remove Protection Domain and Storage Pools from VxFlex OS to remove the protection and storage pools created during the service deployment. If you select this option, you need to select the target VxFlex OS Gateway. The VxFlex OS Gateway will not be removed. VxFlex Manager will only remove the protection domain and storage pools that are part of the service. If multiple services are sharing a protection domain, you might not want to delete the protection domain.

    2. If you are certain that you want to proceed, type DELETE SERVICE.

    3. Click Delete.
