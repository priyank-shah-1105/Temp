Creating credentials

To create credentials:

  1. On the menu bar, click Settings and then click Credentials Management.

  2. On the Credentials Management page, click Create.

  3. In the Create Credentials dialog box, from the Credential Type drop-down list, select one of the following resource types for which you want to create the credentials:

    • Node
    • Switch
    • vCenter
    • Element Manager
    • VxFlex OS Gateway
    • OS

    The OS credential type applies to deployed items, not to the VxFlex Manager appliance itself.

  4. In the Credential Name field, enter the name to identify the credential.

  5. In the User Name field, enter the user name for the credential.

    root is the only valid user name for root-level credentials on nodes (iDRAC). You can add iDRAC users with user names other than root.

    For the OS credential type, you can enter a user other than root. However, on an initial deployment, VxFlex Manager creates only the root user name. Therefore, you'll need to use root, unless you create another user.

    For the VxFlex OS Gateway credential type, you need to provide two separate user names:

    • Gateway Admin User Name
    • Gateway OS User Name

    The Gateway Admin user is the REST API administrator. The Gateway OS user is the Linux SSH login user. The Gateway Admin user must be the admin user and the Gateway OS user must be root.

  6. In the Password and the Confirm Password boxes, enter the password for the credential.

    For the OS credential type, VxFlex Manager will not update the passwords on the node itself. However, if you change the password manually for a user, you can update the password on the Credentials Management page, and VxFlex Manager will ensure that subsequent operations on the service will use the new password.

    For the VxFlex OS Gateway credential type, you need to provide two separate passwords:

    • Gateway Admin Password
    • Gateway OS Password

    VxFlex Manager allows you to supply different passwords for the Gateway Admin and Gateway OS users, but the passwords must be specified correctly. The passwords are validated when you discover the VxFlex OS Gateway on the Resources page, and the discovery will fail if the passwords were not specified correctly on the Credentials Management page. VxFlex Manager will not update the Gateway Admin and Gateway OS passwords on the node itself. However, if you change the password manually for either of these users, you can update the password on the Credentials Management page, and VxFlex Manager will ensure that subsequent operations on the service will use the new password.

    • Optionally, for VMware vCenter and Element Manager, in the Domain box, enter the domain ID.
    • Optionally, for switch credentials:
      1. Under Protocol, click one of the following connection protocols used to access the resource from remote:
        • Telnet
        • SSH
      2. Under SNMP Configuration, in the SNMP v2 Community String box, enter the SNMP v2 community string required to access the resource.

  7. To save the credential, click Save.
