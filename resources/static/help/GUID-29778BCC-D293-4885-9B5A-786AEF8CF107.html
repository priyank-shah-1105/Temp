RCM and OS repositories

In RCM and OS Repositories, you can load Release Certification Matrix (RCM) versions and specify a default RCM for compliance checking. You can load multiple RCMs into VxFlex Manager with different OS images included. The OS images for ESXi and VxFlex OS are included with the RCM. When you add an RCM, these images are automatically added to the OS Image Repositories tab. If necessary, you can also create your own OS image repositories.

Use the Manage RCM Versions tab to load RCM versions and specify a default RCM for compliance checking. It displays the following information:

  • State — Displays an icon indicating one of the following states:
    • Available — Indicates that the RCM is downloaded and copied successfully.
    • Pending — Indicates that the RCM download process is in progress.
    • Error — Indicates that there is an issue downloading the RCM.
  • RCM Name — Display the name of the RCM.
  • Source — Displays the share path of the RCM in a file share.
  • View bundles — Displays details about any bundles added for the RCM.
  • Available Actions — Select any one of the following options:
    • Delete
    • Resynchronize

Use the OS Image Repositories tab to create OS image repositories and view the following information:

  • State — Displays the following states:
    • Available — Indicates that the OS image repository is downloaded and copied successfully on the appliance.
    • Pending — Indicates that the OS image repository download process is in progress.
    • Error — Indicates that there is an issue downloading the OS image repository.
  • Repository — Display the name of the repository.
  • Image Type — Displays the operating system type.
  • Source Path — Displays the share path of the repository in a file share.
  • In Use — Displays the following options:
    • True — Indicates that the OS image repository is in use.
    • False — Indicates that the OS image repository is not in use.
  • Available Actions — Select any one of the following options:
    • Delete
    • Resynchronize

You cannot perform any actions on repositories that are in use. However, you can delete repositories that are in an Available state, but not in use and not set as a default RCM.

All the options are available only for repositories in an Error state. The Resynchronize action will only appear when you need to perform a backup and restore of a previous image.

If a new RCM becomes available, the RCM and OS Repositories page displays a notification banner at the top of the screen with the text A new RCM version is available for download. View Details. To the far right of the banner, you should see an Actions menu that gives you the following choices:

  • View details lets you see details about the new RCM version and download it.
  • Hide lets you hide the banner for the new RCM version. This action applies only to the current user and session. If the current user logs off, the banner reappears when this user logs on again. In addition, the banner appears if a different user logs on. The notification banner also displays if another RCM version becomes available.
  • Dismiss 30 days lets you dismiss the banner for this particular RCM version for 30 days. This action applies only to the current user. The banner appears if a different user logs on. The notification banner also displays if another RCM version becomes available.

The new RCM banner will show up only if you have registered with SRS.
