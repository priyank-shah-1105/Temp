OS Settings

Host Name Selection

If you choose Specify At Deployment Time, you need to type the name for the host at deployment time.

If you choose Auto Generate, VxFlex Manager displays the Host Name Template field to allow you to specify a macro that includes variables that will produce a unique host name. For details on which variables are supported, see the context-sensitive help for the field.

If you choose Reverse DNS Lookup, VxFlex Manager will assign the host name by performing a reverse DNS lookup of the host IP address at deployment time.

OS Image

Specifies the target repository where the OS image install files are located. The default repositories are ESXi.  The additional repositories are shown if you created them in VxFlex Manager.

NOTE: If you select an operating system from the OS Image drop-down menu, the field NTP Node appears. This is an optional component, but we highly recommend that you enter an NTP node IP to ensure proper time synchronization with your environment and VxFlex Manager. Sometimes when time is not properly synchronized, service deployment failure can occur.

Administrator password

Enter the administrator password that set on the installed OS.

Confirm administrator password

Enter to confirm the administrator password.

NTP Node

Specifies the IP address of the NTP node for time synchronization.

If you want to add more than one NTP node in the OS section of a node component, make sure to separate the IP addresses with commas.

OS Credential

Select the credential that you created on the Credentials Management page. Alternatively, you can create a new credential while you are editing a template. If you select a credential that was created on the Credentials Management page, you do not need to type the user name and password, since they are part of the credential definition.

If you upgrade the appliance, you will need to update any older templates and services that included passwords so these templates and services use the correct named credential. After you upgrade the appliance, older templates and services will be in a warning state and the OS Credential field will be empty. To update the template or service, simply select the correct OS credential.

Use Node For Dell EMC VxFlex OS

Indicates that this node component is used for a VxFlex OS deployment. When this option is selected, the deployment installs the MDM, SDS, and SDC components, as required for a VxFlex OS deployment in a VMware environment. The MDM and SDS components are installed on a dedicated VxFlex OS VM (SVM), and the SDC is installed directly on the ESXi host.

To deploy a VxFlex OS cluster successfully, include at least three nodes in the template. The deployment process adds an SVM for each hyper-converged node. VxFlex Manager uses the following logic to determine the MDM roles for the nodes:

  1. Checks the VxFlex OS Gateway inventory to see how many primary MDMs, secondary MDMs, and tie breakers are currently present, as well as the total numbers of SDS components.
  2. Adds the number of components being deployed to determine the overall VxFlex OS cluster size. For example, if there are three SDS components currently in the VxFlex OS Gateway inventory, and you are deploying two more, then you will have a five node cluster after the deployment.
  3. Adds a single primary MDM and determines how many secondary MDMs and tie breakers there should be in the cluster by looking at the overall cluster size. The configuration varies depending on the size of the cluster:
    • A three node cluster has one primary, one secondary, and one tie breaker.
    • A five node cluster has one primary, two secondaries, and two tie breakers
  4. Determines the roles for each of the new components being added, based on the configuration outlined above, as well as the number of primary, secondary and tie breakers that are already in the VxFlex OS cluster.

At deployment time, VxFlex Manager automatically sets up the DirectPath I/O configuration on each hyper-converged node. This makes the devices available for direct access by the virtual machines on the host and also sets up the devices to run in PCI passthrough mode.

For each SDS in the cluster, the deployment adds all of the available disks from the nodes to the storage pools created.

For each compute only or hyper-converged node, the deployment installs the SDC VIB.

VxFlex Manager supports VxFlex OS cluster deployments on ESXi 6.0 and 6.5 nodes.

When you select this option, the teaming and failover policy for the cluster are automatically set to Route based on IP hash. In addition, the uplinks are configured as active and active, instead of active and standby. Teaming is configured for all port groups, except for the VxFlex OS Data 1 and VxFlex OS Data 2 port groups.

If you select the option to Use Node For Dell EMC VxFlex OS, the Local Flash storage for Dell EMC VxFlex OS option is automatically selected as the Target Boot Device under Hardware Settings.

VxFlex OS Role

Specifies one of the following deployment types for VxFlex OS:

  • Compute Only indicates that the node will only be used for compute resources
  • Storage Only indicates that the node will be only used for storage resources
  • Hyperconverged indicates that the node will be used for both compute and storage resources

The Add Existing Service feature only supports the hyper-converged and compute only roles.

If you select an ESXi image type in the OS Image field, the VxFlex OS Role must be set to Compute Only or Hyperconverged. If you add a compute only node, only the SDC is added. If you add a hyper-converged node, both the SDC and SDS are added.

If you select a RHEL image type in the OS Image field, the VxFlex OS Role must be set to Storage Only. If you add a storage only node, only the SDS is added. The only prerequisites for a storage only node are that the iDRAC must have an IP address and a credential. VxFlex Manager takes care of all other configuration steps required for the node. For each node, VxFlex Manager configures the MDM roles as needed and configures the SDS RPMs. Once the cluster has been set up, VxFlex Manager adds every node as an SDS. Then, it adds all available disks for the SDS device, adds a storage pool, and adds all disks to the storage pool.

If you are creating a compute only or hyper-converged template, be sure to include both the VMWare Cluster and VxFlex OS Cluster components in the template builder. If you are creating a storage only template, do not include a VMWare Cluster component in the template builder. Only the VxFlex OS Cluster component is required for a storage only template.

Hardware Settings

Target Boot Device

Specifies the target boot device.

The Local Flash storage for Dell EMC VxFlex OS option is automatically selected as the target boot device if you select the option to Use Node for Dell EMC VxFlex OS under OS Settings.

The target boot device parameter assists in determining which nodes can be selected for deployment in the service. VxFlex Manager filter out nodes that do not meet the requirements to support local flash storage.

If you choose Local Flash Storage, only nodes with a B0SS storage controller and two attached physical disks (Dell EMC 14th generation PowerEdge servers) or SATADOM (Dell EMC 13th generation Dell PowerEdge servers) are selected to be deployed as part of the service.

For 14th generation Dell servers that support BOSS, during deployment VxFlex Manager creates RAID 1 with the two physical disks attached to the BOSS controller.

Node Pool

Specifies the pool from which nodes are selected for the deployment.

BIOS Settings

System Profile 

Select the system power and performance profile for the node.

User Accessible USB Ports 

Enables or disables the user accessible USB ports.

Number of Cores per Processor

Specifies the number of enabled cores per processor.

Virtualization Technology

Enables the additional hardware capabilities provided by virtualization technology.

Logical Processor

Each processor core supports up to two logical processors. If enabled, the BIOS reports all logical processors. If disabled, the BIOS reports only one logical processor per core.

Execute Disable

Enables or disable execute disable memory protection.

Node Interleaving

If the system is configured with matching memory, enables memory node interleaving. If disabled, the system supports nonuniform memory architecture memory configurations.

Network Settings

Add New Interface

Click Add New Interface to create a network interface in a template component. Under this interface all network settings are specified for a node. This interface is used to find a compatible node in the inventory. For example if you add Two Port, 10 gigabit to the template, when the template is deployed VxFlex Manager matches a node with a two port 10-gigabit network card as its first interface.

For more information on adding a new interface, see Adding a new interface.

After entering the information about PXE network in the respective field as described in the table above, VxFlex Manager untags vLANs entered in the PXE network on the switch node facing port. For vMotion and hypervisor network, VxFlex Manager tags these networks on the switch node-facing ports for the entered information. For rack node, VxFlex Manager configures the vLANs on TOR node facing ports (untag PXE vLANs, and tag other vLANs).

If you select Import Configuration from Reference Node, VxFlex Manager imports basic settings, BIOS settings, and advanced RAID configurations from the reference node and allows you to edit the configuration. Some BIOS settings might no longer apply once new BIOS settings are applied. VxFlex Manager does not correct these setting dependencies. When setting advanced BIOS settings, use caution and verify that BIOS settings on the hardware are applicable when not choosing Not Applicable as an option. For example, when disabling SD card, the settings for Internal SD card redundancy become not applicable.

You can edit any of the settings visible in the template, but keep in mind that many settings are hidden when using this option. For example, only ten out of many BIOS settings that you can see and edit using template appear. However, you can configure all BIOS settings. If you want to edit any of the settings that are not visible through the template feature, edit them before importing or uploading the file.

Validate settings shows which potential nodes in inventory are compatible with the component configuration in the template.
