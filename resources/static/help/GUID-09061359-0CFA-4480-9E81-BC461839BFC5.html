Viewing service details

The Service Details tab displays the state of the service at the component level.

At the top of the Service Details page, you can view the topology of the components and connections in a selected service template. If the service state is Incomplete, you will see a banner at the top of the page indicating that you need to add volumes to the service to make it fully functional. For more information on service states, see Service states.

The color of the component icons indicates the following:

Icon colorIndicates
RedThe service is not deployed on a particular component due to some issues.
GreenThe service is successfully deployed on the components.
Light blueThe service deployment is in progress.
YellowThe component requires a firmware update.

Select a component to view more details about the component and, optionally, perform one of the following actions:

  • Click View Logs to display a log of all operations performed on the selected component.
  • Click Port View to display the port view for the selected component.
  • Click Enter Service Mode to put a node in service mode. This action places the SDS in maintenance mode in VxFlex OS and powers down the SVM. It also places the host in maintenance mode in ESXi.

    This button is enabled only when three or more nodes exist in a service. If fewer than thre nodes exist in a service, the button is disabled.

    Only one node can be put in service mode at a time within a service . Once a node in the service is in service mode, the Enter Service Mode button is disabled for all other nodes.

    When you click this button, VxFlex Manager shows a yellow warning banner at the top of the service page and displays the Service Mode icon for the Overall Service Health and for the Resource Health for the selected node. In addition, VxFlex Manager changes the button label to Exit Service Mode for the selected node to allow you to take it out of service mode when you are ready.

    Once a node is in service mode, VxFlex Manager disables the following buttons: Remove Service, Retry, Add Resources, and Delete Resources.

    The View Firmware Compliance Report button remains enabled, but the Update Resources button on the compliance report is disabled.

When you scroll down on the Service Details page, the following information is displayed based on the resource types in the service:

SectionDescription
ClustersView the following information about the clusters in the VMware vCenter environment: Data Center Name, Cluster Name, Management IP, and Asset/Service Tag.
VxFlex OS GatewayView the following information about the gateways that are part of the service:
  • Component Name
  • System Name
  • Primary MDM IP
  • Management IP
  • Protection Domain

    In VxFlex OS, a protection domain is a logical entity that contains a group of SDSs that provide backup to each other. Each SDS belongs to one (and only one) protection domain. Each protection Domain is, therefore, a unique set of SDSs.

  • Storage Pools

    In VxFlex OS, the storage pools allow the generation of different storage tiers in the VxFlex OS system. A storage pool is a set of physical storage devices in a protection domain. Each storage device belongs to one (and only one) storage pool.

    The deployment of a service results in the creation of a separate storage pool for each type of disk (HDD or SSD) that is found in the nodes that are included with the deployment. The deployment process adds all of the disks from all of the nodes to the appropriate storage pools, based on the expected types for each pool.

  • Volumes

    In VxFlex OS, the SDC exposes the VxFlex OS volumes as block devices to the application that resides on the same node where the SDC is installed.

    The deployment of a hyper-converged or storage only service results in the creation of separate volumes within the storage pools. The volumes are mapped to storage pools according to the type of storage (HDD or SSD) expected for the storage pools.

Physical NodesView the following information about the nodes that are part of the service:
  • Hostname
  • VxFlex OS Management IP
  • iDRAC Management IP
  • OS IP
  • Asset/Service Tag
  • VxFlex OS Mode

    The mode for each node is one of the following:

    • Hyper-converged includes both SDS and SDC components.
    • Storage only includes only the SDS component.
    • Compute only includes only the SDC component.
  • MDM Role

    The MDM role is the metadata manager role. The MDM role applies only to those nodes that part of a VxFlex OS cluster. The MDM role is one of the following:

    • Primary: The MDM in the cluster that controls the SDSs and SDCs. The primary MDM contains and updates the MDM repository, the database that stores the SDS configuration, and how data is distributed between the SDSs in the system. This repository is constantly replicated to the secondary MDMs, so they can take over with no delay.

      Every VxFlex OS cluster has one primary MDM.

    • Secondary: An MDM in the cluster that is ready to take over the primary MDM role if necessary.
    • Tie Breaker: An MDM whose sole role is to help determine which MDM is the primary.
    • Standby MDM: A standby MDM can be called on to assume the position of a manager MDM when it is promoted to be a cluster member.
    • Standby Tie Breaker: A standby node that is prepared to take over as a tie breaker.

Service Information

In the Service Information box on the right, you can view the following information:

  • Overall Service Health—Displays the health of the service. The overall service health is determined by the following:

    • Resource Health — Displays the health monitoring of resources.
    • Compliance—Displays if the resources are firmware or software-compliant. This option is applicable only if you have enabled firmware or software update on the service.
    • Deployment State—Indicates if the deployment of the service completed successfully. For more information on service states, see Service states.

  • Deployed By—Displays the name of the user who deployed the service.

  • Deployed On—Displays the date and time when the service is deployed.

  • Reference Template—Displays the name of the reference template used in the service.

    NOTE: For existing services the name is displayed as User Generated Template and not a template name from the inventory.

  • User Permissions—Displays one of the following:

    • Enabled—Indicates that the permission is granted for one or more standard users to deploy this service.
    • Disabled—Indicates that the permission is not granted for the standard users to deploy this service.

Service Actions

Under Service Actions, you can perform the following tasks:

Click...To...
RetryRedeploy a failed service.
Update InventoryUpdate resources for the service.
Remove Service

Remove a service that is no longer required.

VxFlex Manager supports two types of removal. You can delete the entire service, which deletes the deployment information and makes any required configuration changes for components associated with the service. This includes making destructive changes such as deconfiguring the nodes in the service and powering them down.

Alternatively, you can remove just the deployment information for the service without making any configuration changes to the deployed components.

CancelCancel the service.
View All SettingsView the settings configured on the resources in a service for deployment.

For a VxFlex OS service, the VMware Cluster section of the settings page will show the VxFlex OS Settings section, and the Target VxFlex OS Gateway should specify the gateway included with the service.

In addition, for a VxFlex OS service, the Use Local Storage For Dell EMC VxFlex OS property is set to true within the OS Settings section for any node included. This property is set to true for the nodes in an existing VxFlex OS service deployment, as well as for the nodes in a new VxFlex OS service deployment.

Export to FileExport the service details to a .csv file.
Generate troubleshooting bundleGenerate a compressed file of logs to use for troubleshooting. The following logs are included: VxFlex Manager application logs, alert connector logs, VxFlex OS Gateway logs, and iDRAC Lifecycle logs.
View Compliance ReportView the software or firmware compliance report.

Resource Actions

Under Resource Actions, you can perform the following tasks:

Click...To...
Add ResourcesSelect the type of the resources that you want to add to the service.

If you need to input data for the template that is used to create the running service, then the Upgrade Components button is displayed. If you click the Upgrade Components button, the Update Service Component window is displayed. Fields in the Update Service Component window vary depending on templates. Enter values for all displayed fields. Then, click Save.

Delete Resources Delete resources from a service.

Recent Activity

Recent Activity displays component deployment status and information about the current deployed service.

You can also click the Port View tab to display port view details. For more information on port view, see Port View

If you want to see the current firmware or software repository in use, look at Target RCM Version . To change the RCM, click Change Target RCM.
