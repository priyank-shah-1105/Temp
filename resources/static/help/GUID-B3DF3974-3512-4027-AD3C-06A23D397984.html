Viewing the compliance report

Perform the following steps to view a Release Certification Matrix (RCM) compliance report and perform firmware and software updates. You can update software and firmware components for a new service deployed by VxFlex Manager or an existing service that was not originally deployed by VxFlex Manager.

  1. On the menu bar, click Services.

  2. Select a service to view the compliance report.

  3. In the right pane, click View Details.

    The Service Details page is displayed.

  4. On the Service Details page, click View Compliance Report.

    The Release Certification Matrix (RCM) Compliance Report page is displayed.

    You can also view the compliance status on this page.

  5. Click Firmware Components to view the firmware components.

  6. Click Software Components to view the software components.

  7. To update the non-compliant resources, click Update Resources.

    1. To apply the updates right away, choose Allow VxFlex Manager to perform firmware and software updates now.

    2. Click Apply.

    To prevent data loss, VxFlex Manager performs updates for VxFlex OS nodes within a single service in a serialized workflow. The workflow updates one node at a time. This is necessary because each service is associated with a single protection domain, and a protection domain might not span services.

    During the course of execution, the workflow performs these steps for each node and its associated SVM:

    1. Checks to see if the node is part of the VxFlex OS cluster. This check ensures that the workflow does not attempt to update nodes that are not a part of the cluster.
    2. Finds the SVM for each node that is part of the VxFlex OS cluster by performing a look up within the VxFlex OS Gateway.

      The workflow then turns off the SVM. After turning off the SVM, it puts the ESXi node in maintenance mode.

    3. Runs the update task for the firmware that is specified in the RCM. To perform the update, it installs the required update and reboots the ESXi node.

      To ensure that the ESXi node restarts successfully, it also performs a verification check after the reboot.

    4. Takes the node out of maintenance mode.
    5. Powers on the SVM and calls the VxFlex OS Gateway.

    This process repeats for any additional nodes that require updates. The updates for each node may take 45 minutes or more to complete.

    The RCM update process handles node, BIOS, firmware, and ESXi driver updates automatically. However, update to the VxFlex OS software and to top of rack switches must be performed manually.

  8. If you encounter any errors during the course of performing firmware or software updates, you can view the VxFlex Manager logs for the service to see where the error might have occurred.

    1. On the Service Details page, in the right pane, under Service Actions, click Generate Troubleshooting Bundle.

      This creates a compressed file that contains VxFlex Manager application logs, VxFlex OS Gateway logs, and iDRAC Lifecycle logs. The logs are for the current service only.

      Alternatively, you can access the logs from a VMware console, or by using SSH to log in to VxFlex Manager, if you have SSH enabled.
