Updating firmware

A firmware update on a node that is part of a cluster is successful only if the node is set to maintenance mode. VxFlex Manager sets nodes in a cluster to maintenance mode before updating firmware. To ensure that the node remains in maintenance mode, make sure there are other nodes available in the cluster to host the virtual machines of the node being updated.

A firmware update on a VMware cluster is successful only if the cluster is properly configured for HA. For example, the host system can be set in maintenance mode and the virtual machines can be moved from one host to another in the cluster.

If any nodes are not compliant with VxFlex Manager, you can update the firmware to the minimum recommended level.

  1. On the menu bar, click Resources and then click Update Resources.

  2. On the Apply Resource Updates page, select one of the following options:

    • Allow VxFlex Manager to perform firmware and software updates now — Select this option to update the firmware and software immediately.

      VxFlex Manager applies the firmware updates immediately and then reboots all nodes within the service. It updates nodes belonging to a VMware vSphere cluster one at a time by first putting it into maintenance mode, then performing the firmware update and rebooting the node, and finally bringing the node out of maintenance mode before moving on to the next node.

    • Schedule firmware and software updates — Select this option and then select the date and time to update the firmware.

      VxFlex Manager applies the firmware updates at the selected date and time and then reboots all nodes within the service. It updates nodes belonging to a VMware vSphere cluster one at a time by first putting it into maintenance mode, then performing the firmware update and rebooting the node, and finally bringing the node out of maintenance mode before moving on to the next node.

  3. Click Apply.
