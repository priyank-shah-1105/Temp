Getting started

The Getting Started page guides you through the common configurations that are required to prepare a new VxFlex Manager environment. A green check mark on a step indicates that you have completed the step.

Only administrator users have access to the Getting Started page.

The following table describes each steps:

StepDescriptionFor more information
Step 1: Release Certification Matrix (RCM)Provide RCM location and authentication information for use within VxFlex Manager. The RCM defines the specific hardware components and software version combinations that are tested and certified by Dell EMC for hyper-converged infrastructure and other Dell EMC products. This step enables you to choose a default RCM for compliance or add new RCMs.

Step 1 is enabled immediately after you perform an initial setup for VxFlex Manager.

You can also click Settings > OS and RCM Repositories > Manage RCM Versions to manage RCMs.

 Configuring the Release Certification Matrix (RCM)
Step 2: NetworksEnter detailed information about the available networks in the environment. This information is used later during deployments to configure nodes and switches to have the right network connectivity. The networks that you define in VxFlex Manager are used in templates to specify the networks or VLANs that are configured on nodes and switches for your services.

Step 2 is enabled immediately after you perform an initial setup for VxFlex Manager.

You can also click Settings > Network to define, edit, or delete the existing network.

Defining or editing an existing network
Step 3: DiscoverGrant VxFlex Manager access to resources (nodes, switches, virtual machine managers) in the environment by providing the management IP and credential for the resources that are to be discovered.

Step 3 is not enabled until you perform Step 2.

Discovering resources
Step 4: Add Existing ServiceDiscover VMware clusters that are already deployed in an environment and manage them within VxFlex Manager.

Step 4 is not enabled until you perform Step 2.

Adding an existing service
Step 5: TemplatesCreate a template with requirements that must be followed during a deployment. Templates enable you to automate the process of configuring and deploying infrastructure and workloads. For most environments, you can simply clone one of the sample templates provided with VxFlex Manager and make modifications as needed. Choose the sample template that is most appropriate for your environment.
  • For a hyper-converged deployment, clone one of the following templates:
    • Nodes ESXi 13G
    • Nodes ESXi 14G
  • For a two-layer deployment, clone the Compute Only Nodes ESXi 14G template. Then, clone one of the following storage only templates:
    • Dedicated Storage Only Nodes 14G
    • Shared Storage Only Nodes 13G
    • Shared Storage Only Nodes 14G

Step 5 is not enabled until you perform Step 2.

 For more information on cloning a template, see Cloning a template.

For more information on adding a new template, see Adding a template.

To revisit the Getting Started page, click Getting Started on the menu bar, or from the Settings menu.
