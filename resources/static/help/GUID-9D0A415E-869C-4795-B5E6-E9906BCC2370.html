Logs

VxFlex Manager provides an activity log of user- and system-generated actions to use for troubleshooting activities. By default, log entries display in order of occurrence.

These logs contain information for VxFlex Manager application activities. To see component-level logs, generate a troubleshooting bundle.

Activity logs contain the following information:

FieldValueDescription
SeverityA fatal error occurred while communicating with a managed resource; corrective action is immediately required.
The resource is in a state that requires corrective action, but does not impact overall system health. For example, a discovered resource is not supported.
General information about system health or activity.
The component is working as expected.
Cancelled The deployment is canceled.
CategorySecurityAuthentication failures, operations on VxFlex Manager users, operations on credentials.
Appliance ConfigurationInitial setup, appliance settings, backup and restore.
Template Configuration Operations on service templates.
Network ConfigurationOperations on networks, pools for MAC/IQN/WWPN/WWNN.
Infrastructure or Hardware ConfigurationHardware discovery and inventory.
Infrastructure or Hardware MonitoringHardware health.
DeploymentService template deployment operations.
LicensingLicense updates and expirations.
MiscellaneousAll other issues.
DescriptionDisplays a brief summary of activity.
DateThe time when activity occurred, displayed using the client machine time zone. If there are logs, the time captured when the message is logged is based on the appliance time.
UserUser name from which activity originated.
