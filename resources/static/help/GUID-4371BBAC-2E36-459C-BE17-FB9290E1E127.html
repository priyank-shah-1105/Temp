VxFlex OS settings (for the VxFlex OS cluster component)
Target VxFlex OS Gateway

Select a target VxFlex OS gateway. The VxFlex OS gateway acts an endpoint for VxFlex OS API calls. During the provisioning process, VxFlex Manager connects to the VxFlex OS Gateway and uses the VxFlex OS Gateway APIs to configure all SDS and SDC parameters.

Protection Domain Name

Provide a name for the protection domain. You can automatically generate the name (recommended), or specify a new name explicitly.

A protection domain is a logical entity that contains a group of SDSs that provide backup for each other. Each SDS belongs to only one protection domain. Each protection domain is a unique set of SDSs.

If you choose Compute Only as the VxFlex OS Role for the node component, the VxFlex OS settings do not include the Protection Domain Name field.

New Protection Domain Name

Specify a new name for the protection domain.

If you choose Compute Only as the VxFlex OS Role for the node component, the VxFlex OS settings do not include the Protection Domain Name field.

Protection Domain NameTemplate

If you choose to generate the protection name automatically, VxFlex Manager fills this field with a default template that combines static text with variables for several pieces of the auto-generated name. If you modify the template, be sure to include the ${num} variable to ensure that the name is unique.

For details on the rules for defining a template, see the contextual help that appears when you hover over the field in the user interface.

If you choose Compute Only as the VxFlex OS Role for the node component, the VxFlex OS settings do not include the Protection Domain Name Template field.

Storage Pool Name

Provide a name for the storage pool. You can automatically generate the name (recommended), select from a list of existing storage pools for the selected protection domain, or specify a unique storage pool name. If you choose to automatically generate a name, you are prompted to define a storage pool name template.

Storage pools allow the generation of different storage tiers in the VxFlex OS system. A storage pool is a set of physical storage devices in a protection domain. Each storage device belongs to one (and only one) storage pool.

The number of storage pools created at deployment time depends on the number of disks available.

Storage Pool Name Template

Define a template for generating the storage pool name automatically. VxFlex Manager fills this field with a default template that combines static text with variables for several pieces of the auto-generated name. If you modify the template, be sure to include the ${num} variable to ensure that the name is unique.

For details on the rules for defining a template, see the contextual help that appears when you hover over the field in the user interface.

If you choose Compute Only as the VxFlex OS Role for the node component, the VxFlex OS settings do not include the Storage Pool Name Template field.
