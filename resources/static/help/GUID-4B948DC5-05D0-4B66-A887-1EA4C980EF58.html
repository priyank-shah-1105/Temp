Discovering resources

Only administrator-level users can discover resources.

Before you start discovering resources, gather the IP addresses and credentials associated with the resources. Make sure that both the resources and the VxFlex Manager virtual appliance are connected to the network.

For some resources such as nodes, the default credentials are prepopulated in VxFlex Manager. If the credentials have been changed from the defaults, add a new credential to VxFlex Manager with the new login information.

By default, the operational state for all discovered nodes is Unmanaged. If you want to perform firmware updates or deployments on a discovered node, select the node and change the operational state to Managed.

Before adding an existing service for a VxFlex OS environment, you are required to have your vCenter, VxFlex OS Gateway, and node hosts already discovered in the resource list. Only Managed resources are included in the VxFlex OS service. Any resource discovered as Unmanaged will be missing from the service.

When you add an existing service for a VxFlex OS cluster, VxFlex Manager matches up the hosts, vCenter, and other items it finds with discovered resources in the resource list. If you missed a component initially, you can change your resource inventory as needed later on, and update the service to reflect these changes. If you are missing a component, go back to the resources list, select the component, and mark it as Managed by selecting Change resource state to Managed. Then perform an Update Inventory operation on the service to pull in the missing component.

To discover resources:

  1. On the Welcome page of the Discovery wizard, read the instructions, and click Next.

  2. On the Identify Resources page, click Add Resource Type, and perform the following steps:

    1. From the Resource Type list, select a resource that you want to discover.

      • Node
      • Switch
      • vCenter
      • Element Manager
      • VxFlex OS Gateway

    2. Enter the management IP address of the resources that you want to discover in IP Address Range field. To discover a resource in an IP range, provide a starting and ending IP address.

    3. Select one of the following options from the Resource State list:

      • Managed — Select this option to monitor the firmware version compliance, upgrade firmware, and deploy services on the discovered resources. This is the default option for the Switch, vCenter, element manager, and VxFlex OS Gateway resource types.
      • Unmanaged — Select this option to monitor the health status of a device and the firmware version compliance only. The discovered resources are not available for a firmware upgrade or deploying services by VxFlex Manager. This is the default option for the node resource type
      • Reserved — Select this option to monitor firmware version compliance and upgrade firmware. The discovered resources are not available for deploying services by VxFlex Manager.

    4. (Optional) Select an existing or create a node pool from the Discover into Node Pool list. This option allows you to discover the resources into a selected node pool instead of the global pool (default). To create a new node pool, click the + sign to the right of the Discover into Node Pool box.

    5. Select an existing or create a credential from the Credentials list to discover resource types. To create a new credential, click the + sign to the right of the Credentials box. VxFlex Manager maps the credential type to the type of resource you are discovering. The credential types are as follows:

      • Element Manager
      • VxFlex OS Gateway
      • Node
      • Switch
      • vCenter

    6. The default node credential type is PowerEdge iDRAC Default.

  3. Click Next.

    You might have to wait while VxFlex Manager locates and displays all the resources that are connected to the managed networks.

    To discover multiple resources with different IP address ranges, repeat step 2 and 3.

  4. On the Discovered Resources page, select the resources from which you want to collect inventory data and click Finish.

    The discovered resources are listed on the Resources page.
