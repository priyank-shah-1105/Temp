Adding an existing service

To add an existing service:

  1. On the menu bar, click Services and then click + Add Existing Service.

  2. On the Add Existing Service page, enter a service name in the Name field.

  3. Enter a description in the Description field.

  4. Select the Type for the service.

    The choices are Hyperconverged and Compute Only.

  5. To specify the RCM version to use for compliance, select the version from the Release Certification Matrix (RCM) list or choose Use Flex Manager appliance default catalog.

    NOTE: Changing the RCM version might update the firmware level on nodes for this service. Firmware on shared devices will still be maintained by the global default firmware repository.

  6. Specify the service permissions under Who should have access to the service deployed from this template? by performing one of the following actions:

    • To restrict access to administrators, select the Only VxFlex Manager Administrators option.
    • To grant access to administrators and specific standard users, select the VxFlex Manager Administrators and Specific Standard Users option, and perform the following tasks:

      1. Click Add User(s) to add one more standard user to list displayed.

        To remove a standard user from the list, select the user and click Remove User(s).

      2. After adding the standard users, select or clear the check box next to the standard users to grant or block access to use this template.
    • To grant access to administrators and all standard users, select the VxFlex Manager Administrators and All Standard Users option.

  7. Click Next.

  8. On the Cluster Information page, enter a name for the cluster component in the Component Name field.

  9. Select values for the following cluster settings:

    1. Target Virtual Machine Manager—Select the vCenter name where the cluster is available.
    2. Data Center Name—Select the data center name where the cluster is available.
      NOTE: Ensure that selected vCenter has unique names for clusters in case there are multiple clusters in the vCenter.
    3. Cluster Name—Select the name of the cluster you want to discover.

  10. Click Next.

    The list of all the resources available in the cluster as well as the physical nodes are displayed on the Summary page. If VxFlex OS Gateways are found, these are shown in the VxFlex OS Gateways section.

    If the resources are discovered and in available or reserved state, the Available Inventory displays the components as Yes. An unavailable VxFlex OS Gateway is shown as No.

    VxFlex Manager cannot update RCM firmware and software versions for VxFlex OS clusters that do not have available VxFlex OS Gateways. If expected VxFlex OS Gateways are not shown as available, you might need to discover these gateways and run the wizard again.

    NOTE: VxFlex Manager retrieves the host name value from iDRAC and not the operating system. If the host name field is not updated in iDRAC, an incorrect value might appear in VxFlex Manager. Certain operating systems require additional packages to be installed for iDRAC to update the correct host name.

  11. Review the inventory on the Inventory Summary screen.

    The summary shows all nodes that are available. If a node is not available, the reason may be that this node does not match the Type you selected for the service (hyper-converged or compute only).

  12. Click Next.

  13. On OS Credentials page, select the OS credential you want to use for each node and SVM.

    You can select one credential for all nodes (or SVMs), or choose credentials for each item separately. You can create the OS credentials on the Credentials Management page under Settings.

  14. Click Next.

  15. On the Network Mapping page, review the networks mapped to port groups and make any required edits.

    VxFlex Manager attempts to select the correct network based on the VLAN ID or subnet. If VxFlex Manager finds only one network for a given network type, it selects the network automatically. If it finds more than one, you need to select the network from the Network drop-down list. The OS Install network does not get a VLAN ID, so VxFlex Manager presents you with a list of all the networks. You need to choose the correct OS Install network from the list.

    NOTE: If the OS Install VLAN is not already configured in your environment, you need to add it. This network is required to perform node expansions. This network is typically added during VxFlex Manager configuration.

    If there are any port groups for which you do not want VxFlex Manager to manage access, leave those port groups unselected. If no network selection is made for a particular port group, VxFlex Manager leaves it out of the deployment data and does not add it to nodes added later.

  16. Click Next.

  17. Review the Summary page and click Finish when you are ready to add the service.

    The process of adding an existing service causes no disruption to the underlying hardware resources. It simply causes the software to perform several API calls to gather the information it needs to construct the service. It does not shut down any of the nodes or the vCenter.

After the service is created, you can update firmware, change the firmware baseline, and also add resources to or delete resources from the service.

When you add an existing service for a VxFlex OS cluster, VxFlex Manager matches the hosts, vCenter, and other items it finds with discovered resources in the resource list. If you missed a component initially, you can change your resource inventory, and update the service to reflect these changes. If you are missing a component, go back to the resources list, select the component, and mark it as Managed by selecting Change resource state to Managed. Then, perform an Update Inventory operation on the service to pull in the missing component.

When you deploy an existing service, VxFlex Manager reserves any IPs from vCenter or the VxFlex OS Gateway that it needs. If you later tear down the service, it releases those IPs so they can be reused.

CAUTION: Deleting a resource from an existing service deletes the component from the service and the data center environment.
