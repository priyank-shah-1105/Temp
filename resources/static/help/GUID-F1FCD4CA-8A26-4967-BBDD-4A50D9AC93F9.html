Deleting resources from a service

If you are removing a node that is fulfilling an MDM role, verify that you will have at least two MDMs and one tie breaker in the cluster after the removal of the node. If you remove a node, VxFlex Manager puts a standby node in place of the one you removed.

  1. On the menu bar, click Services.

  2. Select a service from which you want to delete resources.

  3. In the right pane, click View Details.

  4. On the Service Details page, under Resource Actions in the right pane, click Delete Resources.

    The Delete Resources from Service page is displayed.

  5. Select the components that you want to delete and click Delete.

If you delete a node from a service, VxFlex Manager performs the following steps to clean up the underlying components associated with the node:

  1. Reconfigures the MDM cluster, if needed, to ensure that the cluster is in good, working order. This is necessary only if the node is part of an MDM cluster.

    If necessary, VxFlex Manager will reconfigure the MDM role for each node to be a Manager or Tie Breaker when you perform a scale down operation. VxFlex Manager will rebalance the roles as needed for a new service or for an existing service. VxFlex Manager will apply the required role changes for nodes that are within the same service or in a difference service.

  2. Removes the node from the list of SDSs within VxFlex OS, if the node is on the list of SDSs. The node might only be removed from the SDS list if the spare capacity is sufficient, and the environment provides enough space.
  3. Deletes the SVM for the node.
  4. Deletes the virtual networking configuration.
  5. Removes the host from vCenter.
  6. Removes the OS on the node.
  7. Removes the node from the SDC list within VxFlex OS.

After a node is removed from a service, the Service Details page shows the cluster without the removed node, and updates the MDM roles as needed to reflect the new configuration.
