Working with a service from a previous release

You can use a service that was originally deployed in version 3.0.1 of VxFlex Manager. You do not need to recreate the service, unless it was an existing service. If the service was an existing service, you need to perform some steps to migrate the deployment to this release.

VxFlex Manager automatically converts a new service that was deployed in version 3.0.1 to the new model for the current release, so you do not need to recreate the service. VxFlex Manager automatically shows the VxFlex OS Cluster on Service Details page. In addition, VxFlex Manager shows the VxFlex OS Cluster settings on the Deployment Settings page. You may need to change some of the settings for the VxFlex OS Cluster component to allow the service to function properly. In addition, you may to click Upgrade Components if VxFlex Manager has added new required fields to any of the components.

VxFlex Manager does not automatically convert an existing service from version 3.0.1 to this release, so you need to perform the following steps to migrate the deployment:

  1. Remove the deployment information for the existing service without making any configuration changes to the deployed components. To do this, you need to select the service and click Remove Service on the Services page. Select Remove Service as the removal type.
  2. Discover and import the hardware resources for the existing deployment. To do this, you need to click Add Existing Service on the Services page.
