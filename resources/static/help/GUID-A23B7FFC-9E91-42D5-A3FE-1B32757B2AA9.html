Dashboard

The Dashboard displays at-a-glance information about service history, resource overview, appliance overview, and activity logs.

Standard users see only the details of services they have created or for which they have permission.

Service overview

The Service history section displays a graphical representation of the services deployed based on status. The number next to each icon indicates the number of services in a particular state. The services are categorized into the following states:

StatusDescription
HealthyGreen band on the graphic; indicates the service is successfully deployed.
WarningYellow band indicates that resources in a service require corrective action, but does not affect overall system health. For example, the firmware version installed on a resource in the service is not compliant.
CriticalRed band indicates services for which the deployment process is incomplete due to errors.
PendingIndicates that deployment is scheduled for a later time or date.
In ProgressBlue band indicates the service is being deployed.
CancelledIndicates that service deployment is stopped. Resources can be updated and deployment retried, if required.
IncompleteIndicates that the service is not fully functional because it has no volumes associated with it. Click Add Resources to add volumes.
Service ModeIndicates that the service has been placed in Service Mode.

You can monitor the health of a node in a service by viewing the status of the service on the Services page.

NOTE: If a service is in a yellow (or warning) state, it means that one or more nodes is in a warning or failed state.  If a service is in a red (or error) state, it indicates that the service has fewer than two nodes that are not in a failed state. 

To view the status of a failed node component, hover the cursor on the image of the failed component in the service.

To display a list of services in a particular state, click the corresponding color band on the graphic. The following information about the services is listed below the graphical display:

  • Status icon
  • Service name; click to view detailed information
  • Name of the user who deployed the service
  • Date and time when the service was deployed
  • Number of resources used by the service based on the component type
  • Any errors

From the Service History drop-down list, you can select to filter and view service deployments by all deployments, the last ten deployments, the last week, the last month, the last six months, or the last year.

Resource overview

The Discovered Resources section shows the number of discovered nodes and switches.

The Node Health section graphically represents the node health.

StatusDescription
CriticalRed band indicates the number of nodes for which the status is critical due to errors.
WarningYellow band indicates that the number of nodes that require corrective action, but do not affect overall system health. For example, the firmware version installed on a node might not be compliant.
HealthyGreen band indicates the number of nodes that were successfully deployed and are healthy.
Service ModeIndicates that the number of nodes that have been placed in Service Mode.
UnknownIndicates that the number of nodes for which the status unknown.

The Node Utilization in Services section displays a pie chart that shows the percentage of nodes in use (blue band) and the percentage of nodes available for deployment (gray band). To view the exact number in use or available, hover the cursor over each band.

In the Utilization by Node Pool section, each bar represents a node pool. It displays the number of nodes used and available in that node pool.

The Storage section displays a pie chart that shows the percentage of storage disk space in use (blue band) and the percentage of available disk space (gray band). To view the percentage, hover the cursor over each band. The Storage section also provides a count of various VxFlex OS components, such as protection domains, volumes, VxFlex OS data clients, and VxFlex OS data nodes.

In the Utilization by Storage Component section, each bar represents a VxFlex OS Gateway resource. It displays the amount of space used within that resource.

Appliance overview

The Appliance Overview section provides a set of buttons that allow you to perform several of the most commonly used actions within VxFlex Manager, such as deploying a new service, adding a template, adding an existing service, and accessing the Getting Started page. In addition, the Appliance Overview displays a gauge meter that shows the storage used by the appliance. The meter shows a green status as long as the storage usage is below 75 percent. Between 75 percent and 95 percent, the meter shows a yellow status, indicating a warning. Above 95 percent, the meter shows a red status to alert you to a critical storage situation.

Whenever the storage usage is either yellow or red, the appliance also displays a banner at the top of the Dashboard to make you aware that storage usage is approaching a critical state.

The License Information displays when the number of resources managed by VxFlex Manager exceeds the valid license count or the trial license expires.

Activity log

The Activity Log section lists the most recent user and system initiated activities. Click View All to view the activities on the Logs page.

The Dashboard also displays the following information:

  • Discovered Resources—Indicates the number of resources that are discovered in VxFlex Manager.
  • Pending Resources—Indicates that the discovery is in progress for the number of resources displayed.
  • Errors—Indicates that VxFlex Manager is unable discover the number of resources displayed due to some errors.
  • Links to learn more about service deployments and templates.
