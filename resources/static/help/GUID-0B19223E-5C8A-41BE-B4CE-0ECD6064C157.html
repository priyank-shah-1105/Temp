Adding a node to a service

To add a node to a service:

  1. On the Duplicate Node page, add the node to the service:

    1. From the Resource to Duplicate list, select a node.

      A service is mapped to a single protection domain. Within a single protection domain, the nodes must be of the same model type. Therefore, select a node that is of the same type as the other nodes within the service.

    2. In the Number of Instances box, enter the number of nodes instances that you want to add to the service. Click Continue.

      The number of instances is fixed for this action.

    3. Specify the VxFlex OS Storage Pool Spare Capacity setting by choosing one of the following options:

      1. Recommended Spare Capacity <n>% sets the spare capacity to 1 divided by the current number of SDSs in the protection domain, plus the number of nodes that you want to duplicate. For example, if you currently have three SDSs and you want to add one more node instance, the recommended spare capacity is set to 25 percent, based on the formula 1/4.
      2. Current Spare Capacity <n>% sets the spare capacity to 1 divided by the current number of SDSs in the protection domain. For example, if you currently have three SDSs in the protection domain, the current spare capacity is set to 34 percent, based on the formula 1/3, rounded up.

    4. In the Component Name box, enter the name of the corresponding nodes.

    5. In the Host Name field, provide a name for the host. For an existing service that was not deployed by VxFlex Manager, the Host Name Selection option is automatically set to Specify at Deployment Time and you need to type the host name.

    6. In the IP Source box, provide an IP address. For an existing service that was not deployed with VxFlex Manager, the default choice is User Entered IP and the IP settings for each network default to Manual Entry. However, you can change the setting to VxFlex Manager Selected IP.

      The Target Boot Device option is automatically set to Use Local Flash Storage for Dell EMC VxFlex OS for an existing hyper-converged or compute only service that was not deployed by VxFlex Manager.

    7. In the Node Source box, select the node source.

      For an existing service not deployed by VxFlex Manager, the node source defaults to Manual Entry, but you can change it to Node Pool.

    8. In the Node Pool box, select the node pool. You can view all user-defined node pools and the global pool. Standard users can see only the pools for which they have permission.

      For an existing service not deployed by VxFlex Manager, the Node Pool defaults to Global.

    Based on the component type, required settings and properties are displayed automatically and can be edited. For more information specific to the component type settings, see Component Types.

  2. Click Save.

When you redeploy an existing service after adding one or more new nodes, VxFlex Manager scales up the deployment by performing the following steps for each node:

  1. Provisions the new node
  2. Updates the switch configuration
  3. Updates the BIOS configuration
  4. Installs ESXi
  5. Creates the SVM
  6. Adds all disks associated with the new node to the SDS
  7. Reconfigures the storage pool
  8. Adds all disks to the storage pool

If you add a node to a three-node cluster, you will still have a three-node VxFlex OS cluster after the scale up operation. The fourth node is added as a standby node. If a node goes down, the standby node is added to the cluster.

If you scale up the deployment to a five-node cluster, VxFlex Manager updates the cluster to be a five-node VxFlex OS cluster. This means that the cluster will have one primary MDM, two secondary MDMs, and two tie breakers. If you subsequently add more nodes, these additional nodes are added as standby nodes.

As you scale up a deployment, VxFlex Manager takes care of any MDM role changes automatically. If necessary, VxFlex Manager will reconfigure the MDM role for each node to be a Manager or Tie Breaker when you perform a scale up operation. VxFlex Manager will rebalance the roles as needed for a new service or for an existing service. VxFlex Manager will apply the required role changes for nodes that are within the same service or in a difference service.

When you redeploy a service after adding one or more nodes, the Resources page displays the following states:

  • Node resources that are part of the service change from Deployed to Deploying, and then to Deployed after the deployment is complete.

  • The new node changes from Not In Use to Pending. Once the deployment starts, the state changes to Deploying. If the deployment is successful, the state changes to Deployed. If the deployment is not successful, the state changes to Critical.
