Adding a volume to a service

A compute only service consumes volumes provided by a storage only or hyper-converged service. Initially, there are no volumes for a new service. The service is in an incomplete state until you add volumes. Add at least one volume to a service to get it out of the incomplete state. If the service is an ESXi cluster that has VMware High Availability (HA) enabled, you need at least two volumes.

After you create the volumes in a hyper-converged or storage only service, they are added to VxFlex OS, but not mapped. When you add the volumes to a compute only service, VxFlex OS maps the volumes and creates the data stores.

To add a volume to a service:

  1. In the Volume Name field, select Create New Volume to create a new volume now, or select an existing volume. For a storage only service, VxFlex Manager shows only the Create New Volume option. For a compute only service, you can select only an existing volume that has not yet been mapped. For a hyper-converged service, VxFlex Manager shows both options.

    If you are creating a new volume, provide the following additional information:

    1. In the New Volume Name field, type the volume name.

    2. In the Storage Pool drop down, choose the storage pool where the volume will reside.

    3. In the Volume Size (GB) field, select the size in Gigabytes. The minimum size is 8GB and the value you specify must be divisible by 8.

    4. In the Volume Type field, select thick or thin.

      A thick volume provides a larger amount of storage in advance, whereas a thin volume provides on-demand storage and faster setup and startup times.

    5. Click Save.

    The service moves to the In Progress state and the new volume icon appears on the Service Details page. The volume is grayed out, because the service is still in progress. After the deployment completes successfully, the volume shows the check mark and appears in the Storage list on the Service Details page. In addition, the VxFlex OS GUI shows the new volume under the storage pool. For a storage only service, the volume is created, but not mapped. For a compute only or hyper-converged service, the volume is mapped to an SDC. In the vSphere client, you can see the volume in the storage section and also see the hosts mapped to the volume, once the mappings are in place.
