Defining or editing an existing network

Adding the details of an existing network enables VxFlex Manager to automatically configure nodes that are connected to the network.

  1. On the menu bar, click Settings and then click Networks.

    The Networks page opens.

  2. Complete one of the following:

    • To define a network, click Define. The Define Network page opens.
    • To edit an existing network, select the network that you want to modify, and click Edit. The Edit Network page opens.

  3. In the Name field, enter the name of the network.

    • Optionally, in the Description field, enter a description for the network.

  4. From the Network Type drop-down list, select one of the following network types. For more information about network types, see Network Types

    • General Purpose LAN
    • Hypervisor Management
    • Hypervisor Migration
    • OS Installation
    • Hardware Management
    • VxFlex OS Data
    • VxFlex OS Data (SDC Traffic Only)
    • VxFlex OS Data (SDS Traffic Only)
    • VxFlex OS Management

    For a VxFlex OS configuration that uses a hyper-converged architecture with two data networks, you typically have two VxFlex OS data networks defined with the VxFlex OS Data network type. The VxFlex OS Data network type supports both SDC and SDS communications. The VxFlex OS Data network type is used with hyper-converged services.

    For a VxFlex OS configuration that uses a two-layer architecture with four dedicated data networks, you typically have two VxFlex OS Data (SDC Traffic Only) VLANs and two VxFlex OS Data (SDS Traffic Only) VLANs. These network types are used with storage only and compute only services.

    You can edit the network type only if the network is not currently referenced by a template.

  5. In the VLAN ID field, enter a VLAN ID between 1 and 4094.

    NOTE: VxFlex Manager uses the VLAN ID specifically to configure I/O modules to enable network traffic to flow from the node to configured networks during deployment. The VLAN ID can be edited only if the network is not currently referenced by a template.

  6. Select the Configure Static IP Address Ranges check box, and then do the following:

    NOTE: After a network is created, you cannot select or clear this check box to configure static IP address pools.

    1. In the Subnet Mask box, enter the subnet mask.

    2. In the Gateway box, enter the default gateway IP address for routing network traffic.

    3. Optionally, in the Primary DNS and Secondary DNS fields, enter the IP addresses of primary DNS and secondary DNS.

    4. Optionally, in the DNS Suffix field, enter the DNS suffix to append for host name resolution.

    5. To add an IP address range, click Add IP Address Range. In the row that displays, indicate the role in VxFlex OS nodes for the IP address range and then specify a starting and ending IP address for the range. For theRole, select either:

      • SDS or SDC: Default; range is assigned to Storage Data Servers (SDS) and Storage Data Client (SDC) roles.
      • SDC: Range is assigned to SDC role on hyper-converged nodes and compute only nodes.
      • SDS: Range is assigned to SDS role on hyper-converged nodes and storage only nodes

      Repeat this step to add multiple IP address ranges based on the requirement. For example, you can use one range for SDC and another for SDS.

      NOTE: IP address ranges cannot overlap. For example, you cannot create an IP address range of 10.10.10.1–10.10.10.100 and another range of 10.10.10.50–10.10.10.150.

  7. Click Save.
