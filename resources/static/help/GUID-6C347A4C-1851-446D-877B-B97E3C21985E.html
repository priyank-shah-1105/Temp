Deploying a service

Deployment is the automated process of selecting and configuring specific resources based on the requirements outlined in a template using VxFlex Manager's built-in automation workflows. You cannot deploy a service using a template that is in draft state. Publish the template before you use the template to deploy a service.

Deployments will fail if Cisco switches do not have LLDP enabled. Be sure to enable LLDP before attempting to deploy a service. If a deployment fails because LLDP is disabled, you need to enable it on the switch to proceed. In addition, you need to update the inventory in VxFlex Manager for the change to be picked up.

To deploy a service:

  1. On the menu bar, click Services.

  2. On the Services page, click Deploy New Service.

  3. On the Deploy Service page, perform the following steps, and then click Next.

    Services

    1. From the Select Published Template list, select the template to deploy a service.

    2. Enter the Service Name (required) and Service Description (optional) that identifies the service.

    3. If you want to update the firmware and software running on the nodes when you deploy a service that uses this template, select Yes for the Perform RCM updates option. Then, from the Select Repository list, select an RCM or firmware repository.

      NOTE: Changing the firmware repository might update the firmware level on nodes for this service. Firmware on shared devices will still be maintained by the global default firmware repository.

    4. Specify the service permissions for this template under Who should have access to the service deployed from this template? by performing one of the following actions:

      • To restrict access to administrators, select the Only VxFlex Manager Administrators option.
      • To grant access to administrators and specific standard users, select the VxFlex Manager Administrators and Specific Standard Users option, and perform the following tasks:

        1. Click Add User(s) to add one more standard users to list displayed.

          To remove a standard user from the list, select the user and click Remove User(s).

        2. After adding the standard users, select or clear the check box next to the standard users to grant or block access to use this template.
      • To grant access to administrators and all standard users, select the VxFlex Manager Administrators and All Standard Users option.

  4. On the Deployment Settings page, configure the required settings, and click Next.

    VMware Cluster: Cluster Settings:

    1. Optionally override any of the cluster settings specified in the template:
      • Data Center Name shows the data center name specified in the template.
      • New Data Center allows you to provide a new data center name.
      • Cluster Name shows the cluster name specified in the template.
      • New Cluster Name allows you to provide a new cluster name.
      • Cluster HA Enabled enables or disables a highly available cluster. You can either select or clear the check box.
      • Cluster DRS Enabled enables or disables distributed resource scheduler (DRS). You can either select or clear the check box.

    VxFlex OS Cluster: VxFlex OS Settings:

    1. Select one of the following options for VxFlex OS MDM Virtual IP Source:
      • VxFlex Manager Selected IP instructs VxFlex Manager to select the virtual IP addresses.
      • User Entered IP allows you to specify the IP address manually for each VxFlex OS data network that is part of the node definition in the service template.

    :

    1. For OS Settings, select one of the following options:
      • VxFlex Manager Selected IP: Indicates the IP of the appliance selected by default.
      • User Entered IP: Indicates the IP of the appliance selected by you. Select the required options for the fields that appear. If you choose User Entered IP, you have the following options for the networks:
        • VxFlex Manager Selected IP
        • Hostname DNS Lookup (is only an option if Reverse DNS Lookup was not selected as the hostname when building the template)
        • Manual Entry

          If you choose Manual Entry, you need to enter the IP address in the static IP address box.

    2. For SVM OS Settings, select one of the following options:
      • VxFlex Manager Selected IP: Indicates the IP of the appliance selected by default.
      • User Entered IP: Indicates the IP of the appliance selected by you. Select the required options for the fields that appear. If you choose User Entered IP, you have the following options for the networks:
        • VxFlex Manager Selected IP
        • Manual Entry

          If you choose Manual Entry, you need to enter the IP address in the static IP address box.

    3. For Hardware Settings:
    • From the Node Source list, select one of the following options:
      • Node Pool: Select the node pool from the list.
      • Manual Entry: Select the node from the Choose Node list.

    Compute Only Node and Storage Only Node:

    1. For OS Settings, select one of the following options:
      • VxFlex Manager Selected IP: Indicates the IP of the appliance selected by default.
      • User Entered IP: Indicates the IP of the appliance selected by you. Select the required options for the fields that appear. If you choose User Entered IP, you have the following options for the networks:
        • VxFlex Manager Selected IP
        • Hostname DNS Lookup (is only an option if Reverse DNS Lookup was not selected as the hostname when building the template)
        • Manual Entry

          If you choose Manual Entry, you need to enter the IP address in the static IP address box.

    2. For Hardware Settings:
    • From the Node Source list, select one of the following options:
      • Node Pool: Select the node pool from the list.
      • Manual Entry: Select the node from the Choose Node list.

    NOTE: You can manually enter the IP address only for a static network.

  5. On the Schedule Deployment page, select any one of the following options:

    • Deploy Now—Select this option to deploy the service immediately.
    • Deploy later—Select this option to enter the date and time to deploy the service later.
