Configuring a Release Certification Matrix

The user interface displays the following information about the firmware repositories:

  • State — Displays the following states:
    • Available — Indicates that the Release Certification Matrix (RCM) is downloaded and copied successfully.
    • Pending — Indicates that the RCM download process is in progress.
    • Error — Indicates that there is an issue downloading the RCM.
  • RCM Name — Displays the name of the RCM.
  • Source — Displays the share path of the RCM in a file share.
  • Available Actions—Select any one of the following options:
    • Delete—Select this option to delete the RCM.
    • Resynchronize—Select this option to resynchronize an RCM that is in an Error state.

Select the RCM to view the following information about the firmware package:

  • Bundles — Displays the number of bundles available in the firmware package.
  • Components—Displays the number of software components available in the firmware package.
  • Created On—Displays the date when the RCM is created.
  • Last Updated—Displays the date when the RCM was last updated.
  • Services Affected—Displays the services in which the firmware is used.

From this page, you can:

  • Add a new RCM.
  • Select Default Release Certification Matrix (RCM) and choose an RCM from the drop-down list to set it as the default for compliance.
  • Select an RCM from the list, and click the delete icon in the same row to remove the RCM.

    If you remove an RCM, the RCM is deleted from the appliance not from the file share.

    The embedded RCM may not be removed.

  • Select an RCM from the list, and, in the right pane, click View Bundles to view the firmware and software bundles available in the RCM.
