Services

A service is an object in VxFlex Manager representing the infrastructure configured during a deployment that persists in VxFlex Manager as long as the infrastructure is required. A service is viewed and interacted with to perform compliance, health, and lifecycle management on the deployed infrastructure. VxFlex Manager supports the deployment of three types of VxFlex OS services:

  • Hyper-converged
  • Storage only
  • Compute only
NOTE: For standard users, only the details of the services they have created or for which they have permission are displayed.

The Services page displays the services that are in the following states in both Tile and List view.

  • Healthy — Indicates that the service is successfully deployed and is healthy.

  • Warning — Indicates that one or more resources in the service requires corrective action.

  • Critical — Indicates that the service is in a severely degraded or non-functional state and requires attention.

  • Pending — Indicates that the deployment is scheduled for a later time or date.

  • In Progress — Indicates that the service deployment is in progress.

  • Cancelled — Indicates that the service deployment has been stopped. Resources can be updated and deployment retired, if required.

  • Incomplete — Indicates that the service is not fully functional because it has no volumes associated with it. Click Add Resources to add volumes.

  • Service Mode — Indicates that the service is in service mode.

On the Services page, you can:

  • Click Deploy New Service to deploy a new service. For more information on deploying a new service, see Deploying a service.
    NOTE: Standard users are allowed to deploy only those services they have created or for which they have permissions.
  • Click Add Existing Service to add an existing service. For more information on adding an existing service, see Adding an existing service.
  • Click Export to File to export all the service details to .csv file.

To switch between List and Tile view, click the Graphic icon or Tabular icon .

To view the services based on a particular service state, select one of the following options from the Filter By drop-down list. Alternately, in the Graphical view, click the graphic in a particular state.

  • All

  • Healthy

  • Warning

  • Critical

  • Pending

  • In progress

  • Cancelled

  • Service Mode

In the Tile view, each square title represents a service and has the status of the service at the bottom of the graphic. The state icon on the graphic indicates the state of the service. The components in blue indicate the component types that are included in the deployment. The components that are in gray indicate the component types that are not included in the service.

In the List view, the following information is displayed:

  • Status — Indicates the status of the service.

  • Name — Indicates the name of the service.

  • Deployed By — Indicates the name of the user who deployed the service.

  • Deployed On — Indicates the date and time when the service is deployed.

  • Components— Indicates the components used in the service.

Click the service in the List view and Tile view to view the following information about the service in the right pane:

  • Service name and description to identify the service.

  • Name of the user who deployed the service.

  • Date and time when the service is deployed.

  • Displays the name of the reference template used in the service.

  • Lists the number of resources included in the service for deployment, based on the following component types:

    • Cluster

    • Node

  • Click View Details to view more details about the service.
  • Click the service name in the List view to open the Service Details page.
  • Click Update Resources to update the firmware of all nodes in the service that are not compliant.
  • Click Export Selected to export the specific service details to .csv file.
