Adding an RCM

  1. From the Getting Started page, click Configure RCM under Step 1: Release Certification Matrix (RCM), then click Add RCM.

    Alternatively, click Settings, then choose OS and RCM Repositories. On the OS and RCM Repositories page, select the Manage RCM Versions tab, and click the Add RCM button.

  2. In the Add Release Certification Matrix (RCM) dialog, select one of the following options:

    • Load RCM from configured Dell EMC Secure Remote Services (Recommended) — Select this option to import the RCM that contains the firmware bundles you need.
    • Load repository from local network path — Select this option to upload the repository from an NFS or CIFS file share. This option is intended for sites that do not have Internet accessibility to Dell EMC Secure Remote Services.

  3. If you selected Load RCM from configured Dell EMC Secure Remote Services (Recommended), click the Available RCMs drop-down list, and select the RCM.

    Before downloading an RCM from Secure Remote Services, you need to configure the alert connector. To configure the alert connector, navigate to Settings, and make changes within the Virtual Appliance Management page.

    If you are downloading an RCM from Secure Remote Services, the file type is a zip or tar.gz file. The xml file type is not supported as an RCM download option for Secure Remote Services.

  4. If you selected Load repository from local network path, perform the following:

    • In the File Path box, enter the location of the RCM file. Use one of the following formats:

      • NFS share for xml file: host:/share/filename.xml

      • NFS share for zip file: host:/share/filename.zip

      • NFS share for tar.gz file: host:/share/filename.tar.gz

      • CIFS share for xml file: \\host\share\filename.xml

      • CIFS share for zip file: \\host\share\filename.zip

      • CIFS share for tar.gz file: \\host\share\filename.tar.gz

    • If using a CIFS share, enter the User Name and Password.

  5. Optionally, set this RCM as the default by choosing Make this default RCM for compliance checking.

  6. Click Save.

    VxFlex Manager unpacks the RCM file as a repository that includes the firmware and software bundles needed to perform any updates.

    If the RCM contains an embedded ISO file for an OS image (such as for an ESXi or VxFlex OS image), the download process unpacks the file and adds it to the OS image repository.
