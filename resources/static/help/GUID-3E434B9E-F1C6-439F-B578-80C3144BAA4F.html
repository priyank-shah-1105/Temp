Adding a new interface

Use the Add New Interface feature to create a network interface to match to a network card on the node when deploying a template.

To add a new interface:

  1. On the node component page, under Network Settings, click Add New Interface. A new interface section appears.

  2. Enter the following information for the new interface:

    • Port Layout — Select the NIC type from the drop-down menu.

      For a 14th generation Dell server ESXi deployment with a Mellanox card, you need to select the two port, 25 gigabit NIC type.

    • Redundancy — Select the Duplicate port settings and configure teaming to enable duplicating port to create redundancy.

  3. Enter the following information for each port:

    • Network (vLAN) — Select the available network to use for data transmission.

      If you choose to use one of the default sample templates, you will see that the positioning of networks on the NICs is laid out for you already.

  4. To view the list of nodes that match the network configuration parameters, click Validate Settings.

    The list of nodes are filtered according to the target boot device and NIC type settings specified.

    When you enable VxFlex OS settings for the node, the Validate Settings page filters the list of nodes according to the supported storage types (All Flash and HDD). Within the section for each storage type, the nodes are also sorted by health, with the healthy (green) nodes appearing first and the critical (red) nodes appearing last.

    The identifying characteristics for each storage type are as follows:

    • Flash nodes have a RAID controller that is HBA330, and all drives connected to it are of type SSD. These nodes are prioritized to the top of the list.
    • All HDD nodes have a RAID controller that is H730 or H740, and have disks connected that are HDD, and have <=2 SSD drives. These nodes appear last in the list.
NOTE: If you select the same network on multiple interface ports or partitions, VxFlex Manager creates a team or bond on systems with the ESXi operating system to enable redundancy.
